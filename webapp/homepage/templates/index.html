Operator - List View

This module provides a visual representation of operators in a list format. Each operator is displayed with an avatar and a brief message.

Module Structure

  1. Operator Card List
    • Operator Avatar Container

      Contains the avatar image and text representation of the operator.

    • Operator Message

      A brief message or description associated with the operator.

Styling and Fonts

The module utilizes custom fonts for enhanced typography:

  • Kantumruy Pro - Used for general text styling.
  • Judson - Used for headings and emphasis.

Assets

  • Operator Avatar Image: Located at /static/operator_circle.svg
  • CSS Stylesheet: Located at /static/css/abop.css

Usage

This module is intended to be used in applications where a list of operators needs to be displayed, allowing users to quickly identify and understand the roles of different operators.

Operator - List View Module

This module is designed to dynamically fetch and display a list of operators using HTMX, enhancing the user experience by loading content asynchronously.

Dynamic Content Loading

The operator list is populated using HTMX, which allows for seamless content updates without requiring a full page reload. The list is fetched from the endpoint /orchestrators when the page loads.

HTMX Integration

  • HTMX Library: The module includes the HTMX library for handling AJAX requests and updating the DOM. It is loaded from a CDN:
    • <script src="https://unpkg.com/htmx.org@2.0.2" integrity="sha384-Y7hw+L/jvKeWIRRkqWYfPcvVxHzVzn5REgzbawhxAuQGwX1XWe70vji+VSeHOThJ" crossorigin="anonymous"></script>

HTML Structure

The main structure of the module consists of an unordered list that serves as a container for the operator cards:

  • Operator Card List: The <ul> element with the class operator-card-list is where the operator cards will be rendered.
  • HTMX Attributes: The hx-get attribute specifies the URL to fetch the operator data, and hx-trigger="load" ensures that the data is loaded when the page is accessed.

Styling and Fonts

The module continues to utilize the same custom fonts for consistency:

  • Kantumruy Pro - Used for general text styling.
  • Judson - Used for headings and emphasis.

Usage

This module is intended for applications that require a dynamic display of operators, allowing for real-time updates and improved performance through asynchronous loading.
